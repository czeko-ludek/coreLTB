## 5. Problem z przekierowaniami (KRYTYCZNY)

### Łańcuch przekierowań

```
http://coreltb.pl/
  → 301 → https://coreltb.pl/
    → 301 → http://www.coreltb.pl/   ← PROBLEM! HTTPS → HTTP
```

**Konsekwencje:**
- Google traci PageRank przy każdym przekierowaniu (~10-15% per hop)
- Wariant http://www.coreltb.pl/ jest canonical, ale nie ma SSL
- Użytkownicy lądują na HTTP — brak kłódki, ostrzeżenie Chrome
- Utrata zaufania (E-E-A-T)

### Wpływ na migrację

Przy migracji MUSIMY:
1. Ustawić `https://coreltb.pl/` jako jedyną wersję canonical
2. Przekierować WSZYSTKIE warianty (http/https × www/non-www) → `https://coreltb.pl/`
3. Zrobić to na poziomie DNS/CDN (Cloudflare), nie na poziomie aplikacji

## 10. Kluczowe wnioski dla migracji

### Silne strony domeny (do zachowania)
1. **506 fraz** w indeksie Google — 12 miesięcy widoczności
2. **Brand queries** generują 39% kliknięć (369/953) — rozpoznawalność marki
3. **7 miast z wyświetleniami** — udowodniona potrzeba na strony lokalne
4. **Homepage z CTR 10.2%** — dobra konwersja z wyników wyszukiwania
5. **Pozycje TOP 10** na frazach lokalne+brand: kierownik budowy katowice (7.2), dom pod klucz jaworzno (4.4)

### Słabe strony (do naprawienia przy migracji)
1. **Brak HTTPS** — natychmiastowy boost po migracji
2. **Thin content** — 200-250 słów na stronach lokalnych (vs 1500-3000 w Elever v3)
3. **75% ruchu na homepage** — zbyt duża koncentracja, strony lokalne nie rankują
4. **46% stron bez ruchu** — martwe URL-e obniżają crawl budget
5. **CTR < 1% na stronach lokalnych** — słabe meta title/description

### Prognozy po migracji
- **Krótkoterminowo (1-3 mies.):** Spadek 10-20% ruchu organicznego (normalne przy migracji)
- **Średnioterminowo (3-6 mies.):** Powrót do poziomu pre-migracji + bonus HTTPS
- **Długoterminowo (6-12 mies.):** Wzrost 2-5x dzięki lepszej treści, HTTPS, Core Web Vitals
